Learning as much as I can about C and Computer Systems

I'm embarking on a journey that interested me for a long time but I was always intimidated by it. I'm going to learn as much as I can about the C programming language and the intersection between hardware and software. I'm doing this to learn more about computers. I wanna understand computers inside out and challenge myself to learn about this incredible machine. It's not gonna be as glamorous as this might sound but I'm up for the challenge.

At first I'm dedicating 40 days of learning and hoping to do it for 10 hours a day and I'm gonna be using a few books that I think will help immensely. These are the books:

  1. C How To Program by Paul Deitel & Harvey Deitel
  2. Problem Solving And Program Design In C by Jeri Hanly & Elliot Koffman
  3. Think Like A Programmer An Introduction To Creative Problem Solving by Anton Spraul
  4. Dive Into Systems A Gentle Introduction To Computer Systems by Suzanne Matthews & Tia Newhall & Kevin Webb
  5. Computer Systems A Programmer's Perspective by Bryant Randal & O'Hallaron David
  6. A Common Sense Guide To Data Structures And Algorithms by Jay Wengrow
  7. Hands On Network Programming With C by Lewis Van Winkle

The first two books for learning C and writing good programs with it. The third for improving my problem solving intuition. The fourth and fifth for learning about computer systems. The sixth for learning algorithms and data structures at a high level, and the seventh one for learning about network programming. All these books use C or a C flavored C++ which is great.

Day 2:

In the 7 hours I studied today I managed to learn about (I used Problem Solving And Program Design In C):

  • recursion at the basic level with two examples: factorial and fibonacci series (I used C How To Program).
  • how to solve problems using the "software development method".
